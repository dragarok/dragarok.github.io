Basic terminologies:

Dimensions Facts

Table and table naming conventions

  • initial 3 letter prefix: DWH, ODS etc.
  • Content: e.g. PRD, SLS
  • Dimension or Fact: D, F
  • Granularity : Department, day/week etc.
  • Suffix as use case: Lookup vs Base Vs Aggregate vs Matrix
  • Naming: data region + dimension_or_fact + content + granularity + usage

Column naming:

  • Naming: Attribute or measure + data type
  • Attribute: Price etc.
  • Data type: KEY, ID, DESC

Maintenance columns: insert timestamp, update, close, close_dt

Business calendar: 4-4-5 or 4-5-4 or 5-4-4 and 13 period calendar.

Matrices are designated with MTX suffix.
