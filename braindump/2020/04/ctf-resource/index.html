9/ Openssl commands:

  1. openssl genrsa -out <output_key_name> : generate a rsa based private public key pair

  2. openssl req -in -text -noout : view certificate signing details

  3. openssl x509 -in <c.cer> -inform -text -noout: examine certificate content : formats can be pem or der

  4. openssl pkcs12 -export -inkey <key_file> -in <cert_file> -certfile <parent_certs> -out <output_name> : get pkcs12 file ; needs key_file, cert_file and parent certificates

  5. openssl x509 -req -in <child_csr> -CA <parent_certificate> -CAkey <parent_key> -CAcreateserial -out <output_child_name> -days <validity_days> -sha256 -extfile : sign child csr with parent

  6. openssl rsa -pubin -in <public_key_file> -text: examine modulus and exponent of RSA public key

  7. openssl pkcs12 -info -in : pkcs12 file details

  8. openssl pkcs12 -in <input_pfx_file> -clcerts -nokeys -out <output_certificate_file> : extract certificate files from pkc file

  9. openssl rsa -in <private_key_file> -text : evaluate rsa private key details: modulus, exponent and so on.

  10. openssl req -addext -outform pem -out <out_csr_name> -key <input_csr_key> -new : generate csr file with extension

Binary forensics:

  3. dd skip=<start_offset> count= if=<input_file> of=<output_file> bs=1 : get a binary file from an existing binary file using offset and count of number of bytes

  4. printf | dd conv=notrunc of= bs=1 seek= : patch a binary with bytes at offset.

    example: printf ‘\x53\xef’ | dd conv=notrunc of=ext-super-magic.img bs=1 seek=$((1024+56)) adds the bytes at the seek position . Useful when changing and adding headers to binary files

step, stepi, next, nexti, finish : i means one instruction and then stop to debugger

step: Continue running your program until control reaches a different source line, then stop it and return control to GDB.

finish(fin): Continue running until just after function in the selected stack frame returns. Print the returned value (if any).
