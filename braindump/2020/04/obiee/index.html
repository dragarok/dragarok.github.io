RMS Sales recon

Tran data

  • In tran_data we have item, units, tran_code as major fields.
  • if it belongs to a pack, then it has a ref_pack_no which is a packitem object.

Packitem

  • It has pack_no and seq_no as main fields: seq_no for multiple items in a pack and make it unique.
  • it has item inside it
  • item_parent if available.
  • either item has to be there or item_parent has to be there. Both can’t be null.
  • pack_qty has quantity of items.

Packitem breakout

  • pack may contain a pack, if so the nested pack and it’s items are inside packitem breakout table.
  • similar to packitem
  • COMP_PACK_NO: Contains the inner pack item number if a pack item is added to an outer pack.
  • COMP_PACK_QTY: This field is null if comp_pack_no is not populated. Else if comp_pack_no is

populated then this field contains the quantity of comp_pack_no in a pack_no.

  • pack_item_qty –> has quantity of each item in pack: if item_parent assoc with pack with no inners directly multiplying pack_qty of item_parent with quantity of item in pack template. if item_parent is assoc to pack that has outer pack –> outer_pack * inner_pack * qty of item in pack template.
