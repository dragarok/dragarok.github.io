Basic rules of complexity definition for removing terms:

  1. Multiplicative constants can be omitted: 14n2 becomes n2.
  2. n^a dominates n^b if a > b: for instance, n^2 dominates n.
  3. Any exponential dominates any polynomial: 3^n dominates n^5 (it even dominates 2^n).
  4. Likewise, any polynomial dominates any logarithm: n dominates (log n)^3. This also means, for example, that n^2 dominates n* log n.

Searching for solution to problem needs you to know first the type of problem it falls into:

  1. Permutations: They are likely the object in question whenever your problem seeks an “arrangement,” “tour,” “ordering,” or “sequence.”

  2. Subsets are likely the object in question whenever your problem seeks a “cluster,” “collection,” “committee,” “group,” “packaging,” or “selection.”

  3. Trees: Hierarchial relations whenever your problem seeks a “hierarchy,” “dominance relationship,” “ancestor/descendant relationship,” or “taxonomy.”

  4. Graphs: relationships between arbitrary pairs of objects Graphs are likely the object in question whenever you seek a “network,” “circuit,” “web,” or “relationship.”

  5. Points: whenever your problems work on “sites,” “positions,” “data records,” or “locations.”

  6. Polygons: working on “shapes,” “regions,” “configurations,” or “boundaries.”

  7. Strings: dealing with “text,” “characters,” ,“patterns,” or “labels.”

Some intuitions behind runtime :

  1. n! will be useless after n > 20

  2. For upto n=10 all algorithms look similar.

  3. 2^n is useless after 40

  4. n^2 is hopeless after a million

  5. linear and nlogn are useful upto a billion

  6. logn is useful for any number that comes up.

Dynamic arrays:

Multiply the memory allocated into 2 times, copy the old one to the lower end of new block and operate.

Main disadv:

Copying the contents of the memory : It will take log2 n doublings until the array gets to have n positions. However, most elements do not suffer much upheaval. Indeed, the (n/2 + 1)st through nth elements will move at most once and might never have to move at all.

Thus, each of the n elements move only two times on average, and the total work of managing the dynamic array is the same O(n) as it would have been if a single array of sufficient size had been allocated in advance!
