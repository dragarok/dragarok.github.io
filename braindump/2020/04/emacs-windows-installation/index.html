Emacs Windows Installation

Problems with installing

  1. Not having msys2 so that pdftools can’t build [install msys]

  2. Not having mingw so that sqlite can’t build for org-roam [choco install mingw] Don’t even have to do that. The mingw is installed by msys2. So go to mysys2: there will be folder named mingw64. Add it’s bin to the PATH.

       pacman -Syu
   pacman -S base-devel
   pacman -S mingw-w64-x86_64-toolchain
   pacman -S mingw-w64-x86_64-zlib
   pacman -S mingw-w64-x86_64-libpng
   pacman -S mingw-w64-x86_64-poppler
   pacman -S mingw-w64-x86_64-imagemagick

  3. Then find a version that supports modules : emax64. This is a well off version and has binaries in different places and things like that because it tries providing it’s own configuration.

  4. Helm-rg didn’t work for me. So, I installed helm-ag and used rg’s executable with it. All is in config file.

  5. Helm-rg will try to say projectile to add itself as well so, don’t use +fuzzy tag with helm.

Some work to do:

  1. Set HOME=“C:\Users\anywhereyoulike”
  2. Add msys2\bin to PATH
  3. Jupyter notebook env directory: $HOME/.conda/
