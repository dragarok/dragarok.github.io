Object Oriented Design

Object orientation is a paradigm that interacts with objects having logic and data by themselves. Object in a real world is similar to the object in programming. The objects have their unique identity. One object might contain other objects but cannot actually replace an object. Multiple attributes are present in an object. Identity sepatate from others. Data to describe it’s current states. Class is a blueprint of the house and house is a object. It can be used to create these objects. Objects are defined as type: methods and attributes Objects are the instances of a class and they are known as instantiation. Existig classes like i/o networking strings dates arrays etc.

Amazing words: a. Abstraction: Nothing specific of an attribute and focus on the essentials. Think of a chair. You can think it anyway . Black Red Small Big or anything . That is known as abstraction : defining the abstract group having similar identity so that the necessary attributes can be added later. Ignores the irrelevant and ignores the unimportant. Similar items will have a same class and under the circumstances and current conditions.

b. Encapsulation: Bundling attributes and methods together in a boundary not just in case but for the protection of the content inside it. Restrict working and sharing of the unnecessary or the inner working of a class when required it gives access to whatever is essential only so that the content is hidden and protected. We can use methods to do a certain task from other part of the application but we restrict in changing the attributes of the object directly without getting inside the object. Black boxing Use the keypad to call dont care about the working of a telephone. Reducing dependencies of the program’s objects and use the encapsulation as much as you can .

c. Inheritance: Code reuse Customer is a person so can do directly customer inherits from the person and add new method or the attributes to be added in the child class. Multiple inheritence can be done in a way that a child class can have more than one parent class from which it inherits it’s attributes and methods. It’s more useful in the sense that it helps us use polymorphism.
