Django Process

FIrst i created project, then created an app Registered the app into the setttings.py file Then first created the view in the app, then add url to the urls.py file in the app If more urls are to be added and multiple apps within the project use include and make urls.py in own app Then made the html file in index directory and worked there. The function in views class takes the request as parameter and returns either HttpResponse or a render object. Render or render_to_response are somehow similar and we give them the request, the file to open i.e. html file and then the variables to be sent with {“temp_name”: my_var}. We edit the html file by putting in the variables as {{ temp_name }} and this works.

TAda… :)

Now use of filters in these variables as {{ temp_name | lower }} makes the variable appear in lowercase More filters are: capfirst, upper Very tasty filter is pluralize: {{ temp_num }} {{ temp_num|pluralize : “y, ies” }} Another one is XSS filter: It helps escape HTML chars. it makes < to &lt , > to &gt and so on. This is done by {% autoescape on %} {% endautoescape %} and {% autoescape off %} and comes the safe filter that makes the variable not escape since it is considered safe inside autoescape on. simply we can make escapes with escape filter. {{ var1 |escape }} linebreaks filter as like
tag

Since we will {% extends base.html %} into index.html, they basically act as one.
