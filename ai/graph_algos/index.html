Breadth first search:

Parent list maintained in bfs can be used to calculate the path to the top. This must be done from bottom towards up.

Connected components: Is a maximal set of vertices such that there is a path between every pair of vertices.

Can be done using discovered nodes list: you take a node, find all that are connected to it via BFS. Then go to unexplored node and do that thing again. Need to have a componentID which marks new component when one BFS finishes.

Bipartite graphs can be verified by making small modification to BFS where we change the color of a node as we process any edge present in graph. If both nodes have same color assigned to them, we can set one node’s color as complement of another node.

Depth First Search:

It uses concept of time as an element in graph.

It uses stack as element for storing nodes to process.

Time changes as we enter or exit any vertex. We keep track of entry and exit time for each vertex.

It can be used without need of stack using recursive implementation.

Set state of node to undiscovered

Process vertex if anything you want to do

Entry time of node is entered into entry list

increment time by 1

for each edge in adjacency list do :

process edge if desired

if state of node with which we have edge is undiscovered then do DFS of the node

Mark the node as Processed

mark exit time in list of exit times

Increment time by 1 again.

Why use time intervals:

Know who is an ancestor in the graph: We cannot exit an ancestor before child in DFS. So, time interval of child must be nested within ancestor.

How many descendants: The difference between entry and exit time for parent divided by two will mark number of descendants.

In case of DFS, we need to be able to do different tasks on entry and exit of a node. So , we have two functions namely process edge early and process edge late for our applications that we require.
